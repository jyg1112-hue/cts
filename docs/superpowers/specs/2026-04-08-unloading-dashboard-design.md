# 하역 데이터 대시보드 설계서

## 1) 목표
- `석탄(년)`/`니켈(년)` 시트를 기반으로 화물/브랜드별 하역 성과를 한 화면에서 비교한다.
- 연도 범위를 사용자가 직접 선택해 누적 실적, 하역률, 이슈를 동시에 확인한다.
- 다크 테마의 경영 요약형 대시보드(참고 이미지 톤)로 가독성을 높인다.

## 2) 범위
- 대상 화면: `unloading_data.html`
- 대상 데이터 소스: `backdata/(2025년) 7선석 하역률.xls`
  - `석탄(년)` 시트
  - `니켈(년)` 시트
- 핵심 지표
  - 하역량(톤)
  - 척수
  - 평균 하역률(시트 `하역율` 원본값 기준)
  - 이슈 건수(`비고` 텍스트 분류)

## 3) 사용자 확정사항
- 화물 축: `석탄`, `니켈`
- 브랜드 규칙
  - 석탄: `품종` 값을 그대로 브랜드로 사용
  - 니켈: 브랜드 사전 매핑 사용 (예: `Ngo(MKM, NMC)` -> `Ngo`)
- 효율 지표: 재계산 없이 시트의 `하역율`만 사용
- 연도 필터: 시작/종료 연도 직접 선택, 기본값은 최근 3개년
- 이슈 카테고리
  - 필수: `돌발정비`, `설비트러블`, `기상불량`, `화물상태`
  - 추가: `작업대기`, `운영변경`, `품질/검출`, `기타`

## 4) 정보 구조(IA) 및 레이아웃

### 4.1 상단 필터 바
- 화물 토글: `석탄 | 니켈`
- 연도 범위: 시작/종료 드롭다운
- 보기 모드: `연도별 | 월별 | 누적`
- 브랜드 멀티셀렉트: 전체/개별 선택

### 4.2 KPI 카드(1행)
- 카드 1: 총 하역량(톤)
- 카드 2: 총 척수
- 카드 3: 평균 하역률(`하역율` 평균)
- 카드 4: 이슈 발생 건수(분류된 이슈 총건수)

### 4.3 메인 시각화(2행)
- 좌측: 월별 복합 차트
  - 막대: 하역량
  - 선: 평균 하역률
  - 연도별 시리즈 비교 지원
- 우측: 이슈 분석 패널
  - 도넛: 카테고리 비중
  - 가로 막대: 카테고리별 발생 건수 TOP

### 4.4 상세 테이블(3행)
- 브랜드 성과 테이블
  - 컬럼: 브랜드, 하역량, 평균 하역률, 척수, 이슈건수, 대표 이슈
  - 기능: 정렬, 검색, 상위/하위 필터

## 5) 데이터 모델 및 변환 규칙

### 5.1 공통 정규화 스키마
- `cargo_type`: `coal`/`nickel`
- `year`, `month`
- `vessel_name`
- `brand`
- `volume_ton`
- `unloading_rate` (원본 `하역율`)
- `remark` (`비고`)
- `issue_tags[]` (분류 결과)

### 5.2 시트별 필드 매핑
- 석탄(년)
  - `volume_ton` <- `하역량(톤)`
  - `brand` <- `품종` (그대로)
- 니켈(년)
  - `volume_ton` <- `B/L량(톤)`
  - `brand` <- `품종`을 브랜드 사전으로 정규화

### 5.3 니켈 브랜드 사전(초안)
- `Ngo(MKM, NMC)` -> `Ngo`
- `Karembe\n(SMT)` -> `Karembe`
- `Poya, Ouaco` -> `Poya/Ouaco`
- 그 외 값은 공백/개행 정리 후 원문 유지

## 6) 이슈 분류 규칙(초안)
- `돌발정비`: "돌발정비", "고장", "파손", "정비" 등
- `설비트러블`: "설비트러블", "비상", "사행", "Trip", "R/S", "R/D" 등
- `기상불량`: "기상", "우천", "한파", "풍", "악천후" 등
- `화물상태`: "화물상태", "수분", "점성", "괴광", "철편" 등
- `작업대기`: "작업대기", "대기", "본선관련", "야드부족", "재고부족" 등
- `운영변경`: "야드변경", "브랜드 변경", "이항양하", "선적조건" 등
- `품질/검출`: "철편검출", "검출", "슈트막힘", "청소" 등
- 매칭 실패: `기타`

> 구현 시에는 정규식 딕셔너리 기반 다중 태깅 후, 화면에서는 대표 카테고리 1개 + 전체 태그 개수를 함께 표기한다.

## 7) 상호작용 설계
- 필터 변경 시 모든 KPI/차트/테이블 동시 갱신
- 차트 클릭(예: 특정 월/카테고리) 시 테이블 자동 필터링
- 화물 전환(석탄/니켈) 시 브랜드 목록 동적 변경
- 연도 범위가 좁을수록 월별, 넓을수록 연도별 축 우선 표시

## 8) 비주얼 가이드
- 다크 배경 + 고대비 카드
- 강조색
  - 하역량: 블루 계열
  - 하역률: 오렌지 계열
  - 이슈: 카테고리별 고정 팔레트
- 카드 반경, 얕은 글로우/그림자, 얇은 보더 사용
- 숫자/단위 분리 표기(`1,604,010 t`, `34,500 t/d` 스타일)

## 9) 오류/예외 처리
- 원본 시트 누락/컬럼명 변경 시 사용자 경고 배너 표시
- 숫자 변환 실패 행은 제외하고 로그 카운트 표시
- `비고` 비어있는 경우 이슈 0건 처리
- 필터 결과 0건이면 빈 상태 안내 + 필터 초기화 버튼 제공

## 10) 검증 계획
- 데이터 검증
  - 시트별 총 하역량 합계가 집계 결과와 일치하는지 확인
  - 연도/월 필터 시 척수 및 하역량 재계산 정확성 확인
- 분류 검증
  - 샘플 30건 수작업 라벨과 자동 분류 결과 비교
  - 카테고리 미분류율(`기타`) 모니터링
- UI 검증
  - 기준 해상도(1440px, 1920px)에서 레이아웃 붕괴 여부 확인
  - 다크 테마 대비/가독성 점검

## 11) 구현 순서
1. `unloading_data.html` 레이아웃/컴포넌트 스캐폴딩
2. XLS 파싱 및 공통 정규화 로직 연결
3. 필터 상태관리 + KPI 계산
4. 차트(월별 복합, 이슈 도넛/바) 연결
5. 브랜드 테이블 및 정렬/검색
6. 이슈 분류 딕셔너리 고도화
7. 스타일 폴리싱(참고 이미지 톤 맞춤)

## 12) 비범위(이번 단계 제외)
- 실시간 데이터 스트리밍
- 사용자별 저장 필터
- 복수 파일 업로드 UI
